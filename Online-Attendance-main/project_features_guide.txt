--------------------------------------------------------------------------------
1. PROJECT OVERVIEW
--------------------------------------------------------------------------------
SmartAttend is a state-of-the-art, cloud-connected Progressive Web App (PWA) 
designed to automate and digitize the daily student attendance tracking process 
for educational institutions. It completely replaces traditional paper registers, 
reduces daily lecture attendance time from 10 minutes to just 15 seconds, and 
eliminates proxy attendance (fake attendance) using a highly secure, time-sensitive 
One-Time Password (OTP) validation mechanism.

--------------------------------------------------------------------------------
2. TARGET AUDIENCES
--------------------------------------------------------------------------------
*   Faculty/Teachers: To take secure, automated attendance, view real-time counts, 
    and instantly download daily or cumulative range-based registers (PDF/Excel).
*   Students: To easily and securely mark their presence using their smartphones 
    or laptops in just a single tap via PWA/Web.
*   Administration/HODs: To access accurate, tamper-proof, cloud-backed historical 
    attendance logs for academic audit and final exams clearance.

--------------------------------------------------------------------------------
3. TEACHER DASHBOARD FEATURES (DETAILED WORKING)
--------------------------------------------------------------------------------
*   Smart Session Setup:
    *   Teachers can dynamically input the class Date, Time, Semester (1 to 8), 
        Subject name, and select single or multiple Divisions (A, B, C) using a 
        clean grid form.
*   Secure 4-Digit OTP Generator:
    *   Generates a random, secure 4-digit OTP code at the click of a button.
    *   The OTP is synced in real-time to the Supabase cloud database.
*   15-Second Expiration Timer:
    *   Includes a visual countdown timer (00:15). 
    *   The OTP remains active for strictly 15 seconds. Once the countdown hits zero, 
        the database OTP is automatically reset to a placeholder ('123'), 
        rendering the code completely invalid. Late-comers are automatically 
        logged as absent.
*   Real-Time "Present Students" Live Counter:
    *   Features a dynamic counter displaying the exact number of students present.
    *   It auto-refreshes every 5 seconds in the background without requiring a 
        manual page reload.
    *   The counter is dynamically isolated to only show submissions matching the 
        currently active Subject, Semester, and Division, ensuring accurate counts.
*   One-Click Single-Day PDF Generator:
    *   Generates a beautifully formatted PDF report for that specific lecture.
    *   Includes official headers ("GEC Modasa - Computer Engineering"), faculty 
        name, selected date, subject, semester, and division labels.
    *   Lists all registered students, highlighting present students in GREEN and 
        absent students in RED.
    *   Strictly filters by the selected subject, date, division, and semester to 
        ensure zero mixed data.
*   Advanced Cumulative "Date Range Reports" Section:
    *   Allows selecting a custom Date Range (From Date to To Date) to generate 
        weekly, monthly, or semester-long reports.
    *   Range PDF (Master Register Layout):
        *   Automatically switches page orientation to Landscape (wide layout) for 
            date ranges larger than 5 days.
        *   Formats date columns as tight, readable headers (e.g. "01/05", "12/05") 
            to maximize horizontal space.
        *   Sets strict custom column widths (Roll No = 28, Name = 38) and enables 
            "ellipsis" text-overflow to prevent vertical name squishing.
        *   Dynamically auto-scales tables, font sizes, and cell paddings depending 
            on the number of days, fitting a full month beautifully on A4 pages.
        *   Fills a clean grid showing 'P' (Green) and 'A' (Red) for each date, 
            along with a total present stats column and attendance percentage.
    *   Range Excel (CSV Master Export):
        *   Generates and triggers a standard, high-performance CSV spreadsheet 
            file instantly on the client side.
        *   Format matches the PDF register (Dates as columns, Students as rows, 
            P/A values, Total stats, and percentages).
        *   Opens natively in Microsoft Excel, Google Sheets, or Numbers for 
            zero-paper digital recordkeeping.

4. STUDENT APP & PWA FEATURES (DETAILED WORKING)
--------------------------------------------------------------------------------
*   Secure Login & Account Guard:
    *   Students log in securely using their Roll Number/Enrollment Number.
    *   Details like Student Name, Division, and Semester are safely loaded from 
        local storage so they never have to re-enter details.
*   Smart WebView/In-App Browser Auto-Detection:
    *   Detects if the link was clicked inside social apps like WhatsApp, 
        Instagram, Facebook, or LinkedIn (Webviews/In-App browsers).
    *   Since these browsers block PWA app installation prompts, the system handles it:
        *   Android: Automatically redirects the user using native Android intents, 
            closing the WhatsApp browser and launching the website directly in the 
            default Google Chrome app, allowing instant "Install App" prompt!
        *   iOS (iPhone): Triggers a gorgeous, blurred glassmorphic Toast notification 
            instructing them step-by-step to click the Share icon and select 
            "Open in Safari" to install the app.
*   OTP Validation & Anti-Cheat Logic:
    *   Validates the student's entered OTP against the active database OTP.
    *   Verifies that the OTP belongs to a teacher currently taking a session for 
        the student's exact division and semester.
    *   Multi-Subject Same-Day Clearance:
        *   Checks duplicates in real-time. Students can securely mark attendance 
            for multiple different subjects (e.g. OS, then DS, then Maths) under 
            the same teacher on the same day without getting blocked, while 
            strictly preventing double-submissions for the same lecture.

--------------------------------------------------------------------------------
5. SECURITY & INTEGRITY PROTOCOLS (ANTI-CHEAT)
--------------------------------------------------------------------------------
*   Anti-Proxy Time limit: The 15-second window makes it impossible for students 
    to text the OTP to their absent friends outside the class before it expires.
*   Device Context Check: Attendance can only be entered on devices currently in 
    the active session.
*   Client-side Context Guard: Context menu clicks (Right click) and Developer Tools 
    shortcuts (F12, Ctrl+Shift+I, Ctrl+Shift+J, Ctrl+U) are strictly disabled across 
    the website to prevent script-inspection or hacking attempts.
*   Tamper-proof Database: Attendance rows are permanently stored securely in the 
    Supabase cloud database with precise timestamps (`created_at`).
